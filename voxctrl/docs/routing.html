INPUT & ROUTING / ROUTING

Routing & targets

Targets are named output destinations. Bindings tell voxctrl which target each hotkey should route to.

Delivery types

Type Mechanism Use for
inject AT-SPI2 → wtype → xdotool Default dictation into focused window
clipboard wl-copy / xclip (Linux), Win32 clipboard (Windows) Copy for manual paste
exec subprocess, shell=false CLI agents, one-shot commands
pipe FIFO write O_NONBLOCK Persistent agent processes
socket TCP / Unix domain Remote or containerised agents
file append + optional timestamp Voice journal, meeting notes
dbus signal emission Waybar, Rofi, desktop automation
http HTTP POST request Webhooks, REST APIs
webhook HTTP POST with custom headers Authenticated webhooks, n8n, Zapier

Post-processing pipeline

The following steps are applied in order after Whisper transcription, configurable per target:

  1. Filler word removal (um, uh, hmm, er, ah, ugh, mhm)
  2. Spoken punctuation ("period" → ., "comma" → ,, 20+ mappings)
  3. Auto-format lists (first/second/third ordinals → numbered list)
  4. Snippet expansion (custom shorthand → full text)
  5. Custom vocabulary fuzzy correction (Levenshtein)
  6. Code mode (camelCase, spoken operators)
  7. Optional Ollama LLM rewrite (clean/formal/casual/bullet/concise/custom)
! Agent targets (exec, pipe, socket) should almost always use remove_fillers = true and spoken_punctuation = false. Rewriting alters command semantics.

Common patterns

Agent command pipe

Send voice commands to a CLI agent via FIFO. Strip fillers so the agent gets clean input. Use response_pipe to stream replies back as TTS.

Voice journal

Append timestamped entries to a markdown file. Use Ollama to format each utterance as a concise note.

Shell command execution

Use delivery = "exec" with a command template. The transcript is passed as a literal argument with shell=false to prevent injection.

TCP socket to remote agent

Forward transcripts over TCP for distributed or containerised setups. Set socket_host and socket_port; voxctrl opens a connection per utterance.

HTTP / webhook delivery

Use delivery = "http" for a simple POST, or delivery = "webhook" to attach custom headers (e.g. Authorization: Bearer …). The transcript is sent as the request body.

Config file locations

  • ~/.config/voxctrl/config.json — global settings (managed by Settings UI)
  • ~/.config/voxctrl/targets.toml — output target definitions
  • ~/.config/voxctrl/bindings.toml — hotkey → target bindings
  • ~/.config/voxctrl/backups/ — auto-backups, last 20 kept
ⓘ All three config files are hot-reloaded on save — no restart required.
