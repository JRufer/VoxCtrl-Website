GETTING STARTED / INSTALLATION

Installation

voxctrl is a Tauri desktop app that runs on Linux (Wayland + X11) and Windows 10/11. The recommended install on Linux is the AppImage — a single portable binary with all runtime dependencies bundled.

System Requirements

Linux

Package Required? Purpose
libwebkit2gtk-4.1 Required Tauri WebView shell
libayatana-appindicator3 or libappindicator3 Required System tray icon
wtype Optional Wayland text injection (for inject delivery type)
xdotool Optional X11 text injection (for inject delivery type)
espeak-ng Optional TTS fallback engine
socat Optional Claude Desktop MCP bridge

Windows

Windows 10 (1903+) or Windows 11. WebView2 runtime is pre-installed on Windows 11; on Windows 10 it downloads automatically on first run.

GPU Acceleration

Hardware Backend Setup
NVIDIA (CUDA 11+) whisper.cpp (CUDA) Auto-detected via nvidia-smi — no extra config
AMD RDNA / GCN whisper.cpp (Vulkan) Install Vulkan ICD: sudo pacman -S vulkan-icd-loader
Intel Arc / Iris Xe whisper.cpp (Vulkan) Install Vulkan ICD: sudo apt install libvulkan1 vulkan-tools
CPU only whisper.cpp (CPU int8) Out of the box — no extra steps
Raspberry Pi / Edge Moonshine (ONNX) Select engine.backend = "moonshine" in Settings → Engine

Option A: AppImage (recommended, Linux)

Download the latest AppImage from GitHub Releases.

Quick run

chmod +x voxctrl.AppImage
./voxctrl.AppImage

System install (with desktop entry and udev rules)

bash install.sh

The install script:

  • Copies the AppImage to ~/.local/bin/voxctrl
  • Installs a .desktop file for application launchers
  • Creates udev rules for /dev/input access (required for global hotkeys)
  • Adds the current user to the input group
! After running install.sh, log out and back in for the input group membership to take effect. Until then, global hotkeys will not work.

Option B: Windows

Download the voxctrl_setup.exe or voxctrl.msi installer from GitHub Releases and run it. WebView2 is bundled or auto-downloaded.

Global hotkeys on Windows use Win32 hooks — no special permissions required.

Option C: Build from source

Prerequisites: Rust (stable), Node.js 20+, npm.

git clone https://github.com/jrufer/voxctrl.git
cd voxctrl
npm install
npm run tauri build

The build output is a release binary + AppImage (Linux) or .exe/.msi (Windows).

For development with hot-reload: npm run tauri dev

First Run

On first launch, voxctrl creates ~/.config/voxctrl/ with default config files and opens the Settings window.

  1. Download a model — Settings → Engine → choose a size → click Download. Recommendation: small (~466 MB) for good speed/accuracy; large-v3 (default, ~3.1 GB) for maximum accuracy.
  2. Check your hotkey — a default binding (Super + Space, hold gesture → inject to focused window) is created automatically. Verify in Settings → Hotkeys.
  3. Test dictation — open any text editor, click into the text area, hold Super + Space, speak, release.

Optional Setup

TTS (Piper voices)

Go to Settings → TTS, enable TTS, select a voice, click Download. Voices are stored at ~/.local/share/voxctrl/piper-voices/.

MCP Server (Claude Desktop)

Enable in Settings → Engine → MCP Server. Then add to ~/.config/claude/claude_desktop_config.json: 

{"mcpServers":{"voxctrl":{"command":"socat","args":["STDIO","UNIX-CONNECT:/tmp/voxctrl-mcp.sock"]}}}

Ollama post-processing

Install Ollama, pull a model (ollama pull llama3.2), then enable in Settings → Ollama.
