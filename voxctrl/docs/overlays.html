AI & VOICE / OVERLAYS

Overlays

voxctrl displays a transparent always-on-top HUD while recording. Four built-in styles ship as Svelte 5 components. Switch between them in Settings → Visual tab.

Built-in styles

Style config value Description
Blue Wave (default) blue_wave Animated ocean waves that react to audio level in real time
Voice Card voice_card Bar-graph spectrum with 45 animated bars and a gradient fill
Waveform waveform 48-bar oscilloscope-style display
Pulse pulse Pill-shaped overlay with animated concentric rings

To disable the overlay entirely, set ui.overlay_style = "none" in ~/.config/voxctrl/config.json or toggle it off in Settings → Visual.

Selecting a style

Open Settings → Visual tab → Overlay Style dropdown. Changes take effect immediately — no restart required. The setting is written to config.json as ui.overlay_style.

// config.json excerpt
{
  "ui": {
    "show_overlay": true,
    "overlay_style": "blue_wave"
  }
}

Valid values: "blue_wave", "voice_card", "waveform", "pulse", "none".

Routing indicator badge

Every overlay shows a small label with the active target's human-readable name (e.g. "Focused Window", "Hermes Agent", "Voice Journal"). When you use multiple hotkeys to route to different destinations, the badge changes with each activation. The label comes from the label field on the active target definition in targets.toml.

How it works

The overlay is a dedicated Tauri window (560×160 px, transparent background, always-on-top, click-through). It renders a Svelte component over whatever application is in focus. Mouse and keyboard events pass through to the underlying window.

Audio levels are forwarded from the Rust backend approximately every 50ms as RMS energy (0.0–1.0). Each overlay component uses this to drive its animation.

The overlay is shown automatically when recording starts and hidden when transcription completes, provided ui.show_overlay = true.

Configuration reference

Key Type Default Description
ui.show_overlay bool true Show/hide overlay on recording
ui.overlay_style string "blue_wave" Active overlay style identifier
