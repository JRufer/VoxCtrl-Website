INTEGRATIONS / MCP

MCP server

Built-in Model Context Protocol server. Any MCP-capable AI agent can drive your microphone and speak through your speakers.

Three tools

Tool What it does
transcribe_voice Opens the mic, records the user, returns the transcript
speak_text Queues text for TTS playback through the current voice
get_status Returns whether the mic is open and TTS is playing

Transport

Unix domain socket at /tmp/voxctrl-mcp.sock (Linux) or named pipe \\\\.\\ pipe\\voxctrl-mcp (Windows). Each connection gets its own async handler. Newline-delimited JSON-RPC 2.0 — one JSON object per line, terminated with \n.

Enabling

Settings → Engine → MCP Server → toggle Enable MCP Server. Or set directly in ~/.config/voxctrl/config.json: 

{"mcp": {"server_enabled": true}}

Claude Desktop

Click Register in Claude Desktop in Settings — voxctrl writes the socat bridge config to ~/.config/claude/claude_desktop_config.json automatically. Restart Claude Desktop. The three tools appear in the picker.

To configure manually, add the following to ~/.config/claude/claude_desktop_config.json:

{
  "mcpServers": {
    "voxctrl": {
      "command": "socat",
      "args": [
        "STDIO",
        "UNIX-CONNECT:/tmp/voxctrl-mcp.sock"
      ]
    }
  }
}
ⓘ voxctrl must already be running before Claude Desktop connects. The socket is created at app startup when MCP is enabled.

Protocol handshake

  • Send initialize — receive protocolVersion 2024-11-05 and capability list
  • Send notifications/initialized (no response expected)
  • Send tools/list — receive the three tool definitions
  • Call any tool with tools/call

Behaviour notes

  • While transcribe_voice records, the recording overlay is always shown — the user always has a visual indicator that the mic is live
  • The mic is released automatically once the RMS noise gate detects silence or timeout_seconds elapses
  • voxctrl's full post-processing pipeline is applied before the transcript is returned
  • speak_text returns as soon as the text is queued — does not block until playback finishes
  • Only one transcribe_voice call can be in flight at a time; concurrent calls queue

Response loopback (FIFO)

For agents that generate responses to a named FIFO, set response_pipe on the target. voxctrl reads each line and passes it to the TTS engine. Each line in the FIFO becomes a separate utterance; empty lines are ignored. The listener re-opens the FIFO on EOF so the agent can restart without restarting voxctrl.
