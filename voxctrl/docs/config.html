REFERENCE / CONFIGURATION

Configuration

All settings live under ~/.config/voxctrl/. The Settings UI is a thin layer over these files — they're the source of truth. All three config files are hot-reloaded on save.

Files

File Format Purpose
config.json JSON Global settings — model, audio, UI, MCP, TTS
targets.toml TOML Output target definitions
bindings.toml TOML Hotkey → target bindings
backups/ — Auto-backups of all three (last 20 kept)

Data directory

Models, TTS voices, and other data are stored under ~/.local/share/voxctrl/:

  • ~/.local/share/voxctrl/models/ — downloaded GGUF / ONNX model files
  • ~/.local/share/voxctrl/piper-voices/ — downloaded Piper TTS voice files

Validation

A startup validator catches malformed config.json, targets.toml, and bindings.toml with clear error messages. Errors fall back to defaults rather than crashing the app.

Key settings

Key Default Description
engine.backend "auto" Inference backend: "auto" | "whisper-cpp" | "moonshine"
engine.whisper_cpp.model_size "large-v3" Active Whisper model name
tts.enabled false Master TTS on/off
tts.voice en-us-lessac-medium Active Piper voice
tts.stop_key [KEY_ESCAPE] Global TTS interrupt
tts.response_overlay true Show teal overlay during TTS
mcp.server_enabled false Start MCP server on launch
mcp.record_timeout 15.0 Default recording timeout (s)
ui.overlay_style "blue_wave" Active overlay style: "blue_wave" | "voice_card" | "waveform" | "pulse" | "none"
atspi.injection true Try AT-SPI2 insertText first
