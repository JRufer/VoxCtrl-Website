REFERENCE / CONFIGURATION

Configuration

All settings live under ~/.config/voxctl/. The Settings UI is a thin layer over these files — they're the source of truth. All three config files are hot-reloaded on save.

Files

FileFormatPurpose
config.jsonJSONGlobal settings — model, audio, UI, MCP, TTS
targets.tomlTOMLOutput target definitions
bindings.tomlTOMLHotkey → target bindings
backups/—Auto-backups of all three (last 20 kept)

Data directory

Models, TTS voices, and other data are stored under ~/.local/share/voxctl/:

  • ~/.local/share/voxctl/models/ — downloaded GGUF / ONNX model files
  • ~/.local/share/voxctl/piper-voices/ — downloaded Piper TTS voice files

Validation

A startup validator catches malformed config.json, targets.toml, and bindings.toml with clear error messages. Errors fall back to defaults rather than crashing the app.
